# Onboarding Buddy

A RAG (Retrieval-Augmented Generation) chatbot that helps new employees get up to speed on their team. Built for a Raspberry Pi 4 using fully local embeddings and a local LLM via Ollama — no paid API required.

## What it does

Onboarding Buddy answers questions about the Payments team at a fictional company (Acme Corp) using a set of uploaded team documents. It retrieves relevant context from those documents and generates a conversational response, maintaining chat history across turns.

**Demo questions it can answer:**
- "What is the team currently working on?"
- "What bug fixes were in the last production deployment?"
- "Who should I talk to about the fraud detection system?"
- "What does the team's sprint cycle look like?"
- "What are the known technical debt items?"
- "Who owns the API contracts?"

## Tech stack

| Component | Tool |
|---|---|
| UI | Streamlit |
| Vector database | ChromaDB (local, persisted to disk) |
| Embeddings | Sentence Transformers `all-MiniLM-L6-v2` (local) |
| LLM | Ollama `llama3.2` (local, free) |
| Orchestration | LangChain (LCEL) |

## Setup

### Prerequisites

- Raspberry Pi 4 (2GB RAM minimum, 4GB recommended)
- Raspberry Pi OS (Bookworm)
- Python 3.13
- 128GB SD card recommended

### 1. Install Ollama

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

Ollama starts automatically as a background service after install.

### 2. Pull the LLM

```bash
ollama pull llama3.2
```

### 3. Install Python dependencies

Install PyTorch CPU-only first to avoid pulling in large CUDA packages:

```bash
sudo pip install --no-cache-dir torch --index-url https://download.pytorch.org/whl/cpu --break-system-packages
sudo pip install --no-cache-dir -r requirements.txt --break-system-packages
```

### 4. Ingest documents

Run once to embed the docs and populate ChromaDB:

```bash
cd ~/onboarding-buddy
python3 ingest.py
```

### 5. Start the app

```bash
python3 -m streamlit run app.py
```

Open your browser to `http://localhost:8501`.

## Performance on Pi 4

Response time is 1–3 minutes per query using `llama3.2` on CPU. This is expected. To confirm the app is working and not hung, open a second terminal and run:

```bash
top
```

You should see the `ollama` process using near 100% CPU while a response is being generated.

## Switching models

Edit the model name in `rag.py`:

```python
llm = ChatOllama(
    model="llama3.2",   # change this
    temperature=0.3,
)
```

| Model | Size | Min RAM | Notes |
|---|---|---|---|
| `tinyllama` | 600MB | 1GB | Fast but low quality |
| `llama3.2` | 1.3GB | 2GB | Best balance for Pi 4 |
| `mistral` | 4GB | 6GB | Higher quality, very slow on Pi |

After changing the model, pull it first:
```bash
ollama pull <model-name>
```
