Production Deployment Log

=== DEPLOYMENT: v2.18.1 ===
Date: May 1, 2026, 11:00 PM ET (maintenance window)
Deployed by: Devon Castillo
Reviewer: Marcus Webb
Environment: Production (us-east-1, eu-west-1)
Rollout strategy: Blue/green with 10-minute canary at 5% traffic

BUG FIXES:
- FIX-8201: Fixed race condition in Visa Direct payout status polling — concurrent status
  checks could overwrite a "settled" record with a stale "pending" response.
- FIX-8209: Rate limiter sliding-window counters were not expiring correctly under Redis
  cluster failover; fixed by adding TTL refresh on counter increment.
- FIX-8215: Apple Pay session validation failed for merchants using custom payment domains
  registered after March 2026; updated domain validation logic to pull from merchant config.

STATUS: Deployment successful. No rollback required.
Post-deploy error rate: 0.02% (within SLA). Latency p99: 138ms (baseline: 141ms, improvement).

=== DEPLOYMENT: v2.17.2 ===
Date: April 10, 2026, 9:00 PM ET
Deployed by: Marcus Webb
Reviewer: Devon Castillo
Environment: Production (us-east-1, eu-west-1)

BUG FIXES:
- FIX-8161: Idempotency key TTL was set to 24h globally; reduced to 2h for refund
  endpoints per PCI DSS guidance on replay attack windows.
- FIX-8167: Reconciliation report endpoint returned HTTP 500 when date range spanned a
  DST boundary; fixed timezone normalization in ReportQueryBuilder.
- FIX-8170: Sub-account rate limit quota was shared with parent account when
  sub-account had no explicit limit configured; now defaults to 20% of parent quota.

STATUS: Deployment successful.
