Production Deployment Log

=== DEPLOYMENT: v2.14.3 ===
Date: April 18, 2025, 11:30 PM ET (maintenance window)
Deployed by: Devon Castillo
Reviewer: Marcus Webb
Environment: Production (us-east-1, eu-west-1)
Rollout strategy: Blue/green with 10-minute canary at 5% traffic

BUG FIXES:
- FIX-7821: Resolved race condition in idempotency key cache causing duplicate charge events
  under high concurrency. Root cause was missing lock on Redis SET NX operation.
- FIX-7834: Fixed NullPointerException in PaymentMethodService when Apple Pay token
  response omitted the `billingContact` field (optional per spec but not handled).
- FIX-7850: Corrected timezone handling bug in settlement batch job — batches scheduled
  near DST transitions were running 1 hour early, causing off-cycle reconciliation.
- FIX-7861: Webhook signature verification failed silently when payload exceeded 64 KB;
  now returns HTTP 413 with descriptive error and logs alert.

STATUS: Deployment successful. No rollback required.
Post-deploy error rate: 0.03% (within SLA). Latency p99: 142ms (baseline: 138ms, acceptable).

=== DEPLOYMENT: v2.14.2 ===
Date: April 10, 2025, 10:00 PM ET
Deployed by: Amara Osei
Reviewer: Priya Nair
Environment: Production (us-east-1 only; EU rollout held pending GDPR review)

BUG FIXES:
- FIX-7799: Payment capture incorrectly marked as failed when downstream processor returned
  HTTP 202 (Accepted) instead of 200 (OK). Fixed status mapping in ProcessorResponseAdapter.
- FIX-7803: Fraud scoring service returned cached score for new card BINs not yet in
  the BIN lookup table. Now falls back to conservative default score (0.6) with a
  cache-miss metric emitted for monitoring.
- FIX-7811: Refund endpoint accepted negative amounts without validation, allowing
  unintended credit creation. Added server-side guard (amount must be > 0 and ≤ original charge).

STATUS: Deployment successful. EU rollout completed April 14 after GDPR sign-off from Legal.
